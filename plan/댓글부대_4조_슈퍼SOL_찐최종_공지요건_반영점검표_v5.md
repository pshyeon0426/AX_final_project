# 공지문 및 추가 요건 반영 점검표 찐최종본 v5

# 슈퍼SOL 앱 출시 초기 고객 반응·경쟁 앱 벤치마킹·리뷰 답글 자동 생성 서비스

본 점검표는 신한은행 AX 전문가 Intensive 과정 최종 프로젝트 공지문과 추가 요건인 “실제 리뷰에 대한 OpenAI API 기반 답글 자동 생성기”를 기준으로 PRD v5와 개발 프롬프트 v5를 점검한 결과이다.

---

## 1. 종합 판단

v5에서는 기존 v4의 공지문 반영 사항을 유지하면서, 추가 요건인 리뷰 답글 자동 생성기를 단순 부가기능이 아니라 **CX/VOC 실무 활용 기능**으로 제품 범위에 통합했다.

핵심 보완 사항은 다음과 같다.

1. 제품명을 “고객 반응·경쟁 앱 벤치마킹·리뷰 답글 자동 생성 서비스”로 확장했다.
2. 답글 생성 기능을 PRD의 기능 요구사항, 데이터 설계, 화면 요구사항, 비기능 요구사항, 검증 계획, 일정, 리스크에 모두 반영했다.
3. OpenAI API 키는 `OPENAI_API_KEY` 환경변수로 관리하고 실제 키는 제출물에 포함하지 않도록 명시했다.
4. 답글 생성은 자동 게시가 아니라 담당자 검수용 초안으로 제한했다.
5. 개인정보 요청, 보상/처리 확정 약속, 금융 조언, 고객정보 추정 등 금지 원칙을 추가했다.
6. 답글 품질 검증 산출물 `reply_eval_report.md`, `reply_eval_sample.csv`, `reply_risk_cases.csv`를 추가했다.
7. Streamlit에 “답글 생성 탭”과 “답글 품질 검증”을 추가했다.
8. 발표 Q&A에서 답글 자동 생성의 안전성과 API 키 관리 방식을 설명할 수 있도록 보완했다.

## 5. 개발 시 특히 주의할 점

1. 불만 유형 분류는 MVP 기준 룰 기반이므로 “ML 성능”으로 발표하면 안 된다.
2. 감성분석 F1-score는 별점 기반 약지도 라벨 기준이므로 한계를 반드시 설명해야 한다.
3. baseline 대비 개선이 없으면 실패가 아니라, 미달 사유와 개선 시도를 명확히 보여주는 것이 중요하다.
4. App Store/Google Play 수집이 막혀도 수동 CSV와 샘플 CSV로 데모 가능해야 한다.
5. Streamlit 데모는 인터넷이 끊겨도 실행되도록 샘플 데이터, 사전 생성 리포트, 사전 생성 답글 CSV를 준비해야 한다.
6. 답글 생성 기능은 OpenAI API 키가 없어도 fallback으로 동작해야 한다.
7. 답글은 자동 게시하면 안 되며, 담당자 검수용 초안임을 화면과 발표에서 명확히 말해야 한다.
8. API 키는 `.env` 또는 환경변수로만 관리하고, GitHub/제출 zip에는 절대 포함하지 않는다.
9. 답글 품질 검증에서는 좋은 답변만 보여주지 말고, 부적절 가능성이 있는 사례와 개선 방안도 함께 보여줘야 한다.
10. 발표 Q&A에서는 “왜 이 기술을 썼는가”, “이 결과가 맞다는 근거는 무엇인가”, “답글 자동 생성은 안전한가”, “실무에 어떻게 쓸 수 있는가”를 설명해야 한다.

## 6. 최종 결론

v5 PRD와 개발 프롬프트는 공지문 기준의 필수 요건과 추가 요건을 모두 반영한다. 특히 평가에서 중요한 데이터 처리·분석, 모델/시스템 완성도, 기술 통합도, 발표/Q&A, 금융 업무 활용 가능성을 방어할 수 있도록 산출물과 검증 기준을 구체화했다.

가장 중요한 실행 포인트는 다음 5가지이다.

1. `notebooks/01_EDA_preprocessing.ipynb`에 전처리 판단 근거를 충분히 남긴다.
2. `outputs/model_comparison.csv`로 baseline 대비 개선을 보여준다.
3. `outputs/validation_report.md`에 성공 기준 pass/fail과 실패 케이스를 명시한다.
4. `outputs/reply_eval_report.md`로 답글 생성 품질과 안전성을 검증한다.
5. README와 발표자료에 팀원별 역할, 실행 방법, OpenAI API 키 관리, 구현 이유를 확실히 적는다.
