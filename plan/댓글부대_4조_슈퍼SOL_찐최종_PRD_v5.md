## 1. 제품 개요

신규 금융 앱 출시 초기에는 앱스토어 리뷰, 문의성 댓글, 경쟁 앱 반응이 빠르게 쌓인다. 담당자가 수작업으로 확인하면 주요 불만, 기능 요구, 이탈 징후, 경쟁 앱 대비 약점, 고객 응대 필요 리뷰를 늦게 파악할 수 있다.

본 프로젝트는 공개 리뷰와 경쟁 앱 공개 문서를 수집·정제·분석하여 **고객 감성, 불만 유형, 주요 토픽, 경쟁 앱 강점, 개선 우선순위, 리뷰 답글 초안**을 한 화면에서 확인하는 AI 기반 VOC·벤치마킹·고객응대 보조 서비스이다.

| 핵심 사용자 | 앱 기획자, 디지털 채널 담당자, CX/VOC 담당자, 마케팅 담당자, 고객응대 담당자, 보고 담당자 |

## 2. 문제 정의

### 2.1 해결하려는 문제

슈퍼SOL과 같은 신규 금융 앱은 출시 초기 사용자 반응이 빠르게 축적된다. 앱스토어 별점만으로는 고객 불만의 원인, 기능 요구, 경쟁 앱 대비 부족한 지점, 긴급 응대가 필요한 리뷰를 명확히 파악하기 어렵다. 담당자가 수작업으로 리뷰를 확인하고 답변까지 작성하면 대응이 늦어지고, 보고서 작성 및 반복 응대 업무에도 많은 시간이 소요된다.

### 2.2 업무 영향

| 현재 문제 | 업무 영향 | 해결 방향 |
|---|---|---|
| 리뷰가 여러 앱/스토어에 분산 | 초기 고객 반응 파악 지연 | 앱별/스토어별 리뷰 통합 수집 및 정제 |
| 별점만으로 원인 파악 어려움 | 개선 우선순위 판단 어려움 | 리뷰 본문 기반 감성분석, 불만 유형 분류 |
| 경쟁 앱 장단점 비교가 수작업 | 벤치마킹 근거 부족 | 공개 리뷰·기능 설명 기반 경쟁 비교 |
| 반복적 고객 답글 작성 | 고객응대 속도 저하, 답변 품질 편차 | 리뷰 유형·감성·이슈별 답글 초안 자동 생성 |
| 보고서 작성 반복 | 의사결정 속도 저하 | AI 요약 리포트 자동 생성 |
| 분석 결과 검증 부족 | Q&A에서 신뢰도 저하 | 데이터·모델·RAG·답글 품질·UAT 검증 결과 제시 |

### 2.3 가설

1. 앱 공개 리뷰를 수집·정제하고 EDA를 수행하면 출시 초기 고객 반응과 급증 이슈를 정량적으로 파악할 수 있다.
2. 리뷰 본문을 TF-IDF 기반 머신러닝 모델로 학습하면 긍정/중립/부정 감성을 자동 분류할 수 있다.
3. 토픽 분석과 불만 유형 분류를 결합하면 로그인, 인증, 속도, 혜택, 송금, 투자, UX, 오류 등 개선 우선순위를 도출할 수 있다.
4. 경쟁 앱 공개 문서와 리뷰를 RAG로 검색하면 근거 기반의 벤치마킹 포인트를 제시할 수 있다.
5. 리뷰 감성·불만 유형·심각도·응대 정책을 LLM 프롬프트에 반영하면 고객 응대용 답글 초안을 빠르게 생성할 수 있다.
6. Streamlit 대시보드와 AI 요약 리포트, 답글 초안 생성기를 제공하면 기획·마케팅·CX 담당자가 빠르게 의사결정하고 고객 응대 품질을 표준화할 수 있다.

## 6. 핵심 사용자 흐름

1. 사용자가 Streamlit에서 샘플 데이터 사용 또는 CSV 업로드를 선택한다.
2. 앱 목록, 기간, 스토어, 경쟁 앱을 선택한다.
3. 데이터 검증 모듈이 필수 컬럼, 결측, 중복, 날짜, rating 범위를 점검한다.
4. 전처리 모듈이 리뷰 본문 정제와 개인정보성 패턴 마스킹을 수행한다.
5. EDA 모듈이 리뷰 수, 평점, 부정률, 기간별 추이, 키워드를 생성한다.
6. ML 모듈이 감성분석 baseline과 개선 모델을 비교하고 성능을 산출한다.
7. 불만 유형 분류기가 로그인, 인증, 속도, 혜택, 송금, 투자, UX, 오류 등 유형을 부여한다.
8. 토픽 분석 모듈이 주요 토픽 5개 이상과 대표 리뷰를 도출한다.
9. 경쟁 앱 벤치마킹 모듈이 앱별 평점, 부정률, 불만 TOP5, 기능 키워드, 개선 기회를 비교한다.
10. RAG 모듈이 경쟁 앱 공개 문서에서 근거 Top-k를 검색한다.
11. LLM 또는 rule-based fallback이 핵심 요약, 개선 우선순위, 벤치마킹 포인트, 데이터 한계를 담은 리포트를 생성한다.
12. 사용자가 부정 리뷰 또는 답변 필요 리뷰를 선택하면 답글 생성기가 감성, 불만 유형, 심각도, 금칙어 규칙을 반영해 답글 초안을 생성한다.
13. 담당자는 답글 초안을 검토·수정·복사하고, 필요 시 “고객센터 문의 필요”, “장애 확인 필요”, “개인정보 포함 가능성” 상태를 지정한다.
14. 검증 탭에서 데이터 품질, 모델 성능, RAG 평가, 답글 품질 평가, 실패 케이스, UAT 결과를 확인한다.

## 9. 분석 및 모델 설계

### 9.1 Pandas EDA + 전처리

- 앱별 리뷰 수, 평균 평점, 부정률, 기간별 추이 분석
- 스토어별 리뷰 비중 및 편향 확인
- 결측, 중복, 날짜 파싱 실패, 리뷰 길이 이상치 확인
- 리뷰 길이·반복 문구·동일 문장 중복 등 이상 패턴 처리 기준을 노트북에 명시
- 최종 산출물: `notebooks/01_EDA_preprocessing.ipynb`, `outputs/eda_summary.json`, `outputs/data_quality_report.md`

### 9.2 머신러닝 감성분석

| 단계 | 모델/방법 | 목적 |
|---|---|---|
| Baseline 1 | DummyClassifier 또는 최빈 클래스 예측 | 최소 기준선 |
| Baseline 2 | TF-IDF unigram + Logistic Regression 기본값 | 단순 ML 기준선 |
| Improved | TF-IDF ngram + Logistic Regression/LinearSVC + class_weight/hyperparameter 조정 | 베이스라인 대비 개선 |
| Manual Check | 수동 검수 100~200건 기준 평가 | 약지도 라벨 한계 보완 |

- 라벨 생성: 1~2점 negative, 3점 neutral, 4~5점 positive
- 단, 별점 기반 라벨은 약지도 라벨이므로 실제 감성과 다를 수 있음을 리포트에 명시
- 평가지표: accuracy, precision_macro, recall_macro, f1_macro, confusion matrix
- 목표: macro F1-score 0.75 이상 또는 미달 사유와 개선 방안 제시
- 최종 산출물: `models/sentiment_model.pkl`, `outputs/model_report.md`, `outputs/model_comparison.csv`, `outputs/metrics.json`

### 9.3 불만 유형 분류

- MVP 기준: 키워드 사전 기반 룰 분류
- 유형: 로그인, 인증, 속도, 오류, 혜택, 송금, 투자, UX, 기타
- 한 리뷰가 여러 유형에 해당할 수 있으므로 멀티라벨 리스트로 반환
- 이 기능은 MVP 기준 ML이 아니라 룰 기반임을 발표에서 명확히 설명
- 선택 고도화: 수동 라벨이 충분하면 TF-IDF + OneVsRestClassifier로 멀티라벨 ML 분류 추가

### 9.4 심각도/응대 필요도 분류

답글 자동 생성 기능을 위해 리뷰별 심각도를 간단히 분류한다.

| 심각도 | 예시 | 응대 방향 |
|---|---|---|
| high | 로그인 불가, 결제/이체 실패, 장애 반복, 보안 우려 | 사과 + 불편 인지 + 고객센터/앱 내 문의 안내 + 개인정보 입력 금지 |
| medium | 속도 느림, 메뉴 복잡, 혜택 확인 어려움 | 공감 + 개선 검토 안내 + 사용 경로/문의 안내 |
| low | 단순 문의, 제안, 가벼운 불만 | 감사 + 안내 + 개선 참고 |
| positive | 칭찬, 만족 | 감사 + 지속 개선 약속 |

### 9.5 토픽 분석

- TF-IDF + KMeans 기반 비지도학습
- 토픽 수 기본 5개 이상
- 각 토픽별 대표 키워드 10개, 대표 리뷰 5개 제시
- 토픽명이 애매하면 사람이 수정할 수 있도록 `topic_label` 컬럼 제공

### 9.6 경쟁 앱 벤치마킹

- 슈퍼SOL과 경쟁 앱 3~5개 비교
- 비교 지표: 리뷰 수, 평균 평점, 부정률, 최근 30일 평점 변화량, 불만 유형 TOP5, 주요 토픽 TOP5, 긍정 키워드, 공개 기능 키워드
- 산출물: `outputs/benchmark_summary.csv`, `outputs/competitor_gap_analysis.md`
- 경쟁사 내부 전략은 추정하지 않고 공개 리뷰와 공개 문서 근거로만 작성

### 9.7 RAG/LLM 리포트

- 경쟁 앱 기능 설명, 릴리즈 노트, 공개 기사/공지 문서를 `docs/competitors`에 저장
- ChromaDB 또는 TF-IDF fallback으로 근거 검색
- 테스트 질문 5개에 대해 Top-3 근거를 추출하고 관련성 평가 샘플 생성
- LLM 호출 실패 시 rule-based summary fallback 제공
- 프롬프트 원칙: 근거 없는 추정 금지, 내부 전략 추정 금지, 공개 데이터 기준, 데이터 한계 명시

### 9.8 리뷰 답글 자동 생성

- 입력: 리뷰 원문, 평점, 감성분석 결과, 불만 유형, 심각도, 앱명, 답글 톤, 금칙어/주의사항
- 출력: 고객에게 게시 가능한 수준의 답글 초안, 응대 유형, 안전 플래그, 검수 필요 여부
- 기본 방식: OpenAI API 호출
- fallback: LLM API 키가 없거나 호출 실패 시 룰 기반 템플릿 답글 생성
- API 키 관리: `.env` 또는 환경변수 `OPENAI_API_KEY` 사용. 실제 키는 GitHub/제출파일에 포함 금지
- 모델명은 `OPENAI_MODEL` 환경변수로 분리해 교체 가능하게 구성
- 답글 원칙:
  - 한국어, 공손하고 짧은 톤
  - 리뷰 내용에 공감하고 필요한 경우 사과
  - 개인정보, 계좌번호, 비밀번호, 인증번호 입력을 요청하지 않음
  - 보상, 장애 원인, 처리 결과를 확정적으로 약속하지 않음
  - 고객별 계좌/거래 내역을 알고 있는 것처럼 말하지 않음
  - 금융상품 투자 조언 또는 확정적 추천 금지
  - 심각한 오류/보안/금전 관련 이슈는 고객센터 또는 앱 내 문의로 안내
  - 자동 게시가 아닌 담당자 검수용 초안임을 UI에 표시

| 재현성 | README에 설치, 실행, 샘플 데이터, 테스트 방법, API 키 설정 방법을 명시한다 |
| 해석 가능성 | 모델 결과만 제시하지 않고 주요 키워드, 원문 리뷰, 근거 문서, 답글 생성 근거를 함께 표시한다 |
| 안정성 | 수집 실패, 빈 데이터, 컬럼 누락, API 차단, LLM 호출 실패, API 키 누락 시 사용자에게 대체 안내를 제공한다 |
| 발표 적합성 | 네트워크가 불안정해도 샘플 CSV, 로컬 RAG 인덱스, 사전 생성 리포트, 룰 기반 답글 fallback으로 데모 가능해야 한다 |

## 13. OpenAI API 기반 답글 생성 설계

### 13.1 구성 파일

| 파일 | 역할 |
|---|---|
| `.env.example` | `OPENAI_API_KEY=your_key_here`, `OPENAI_MODEL=your_model_here` 예시 제공 |
| `src/reply_generator.py` | OpenAI API 호출, fallback, 안전 플래그 처리 |
| `prompts/reply_prompt.md` | 답글 생성 시스템 프롬프트와 출력 형식 정의 |
| `config/reply_policy.yaml` | 톤, 금칙어, 이슈별 응대 원칙, 고객센터 안내 문구 관리 |
| `outputs/reply_drafts.csv` | 생성된 답글 초안 저장 |
| `outputs/reply_eval_report.md` | 답글 품질 검증 결과 저장 |

### 13.2 답글 출력 형식

```json
{
  "reply_draft": "고객님, 이용에 불편을 드려 죄송합니다...",
  "reply_type": "apology_and_guidance",
  "safety_flags": ["no_personal_info_request", "needs_human_review"],
  "needs_human_review": true,
  "reason": "로그인 오류 및 계정 관련 가능성이 있어 고객센터 안내 필요"
}
```

### 13.3 답글 유형

| 유형 | 사용 조건 | 기본 톤 |
|---|---|---|
| `thanks_positive` | 긍정 리뷰 | 감사 + 지속 개선 약속 |
| `apology_and_guidance` | 오류/장애/불편 리뷰 | 사과 + 불편 인지 + 문의 경로 안내 |
| `feature_request_ack` | 기능 개선 요청 | 의견 감사 + 담당 부서 전달/개선 참고 |
| `how_to_guidance` | 사용 방법 문의 | 간단 안내 + 상세 문의 경로 |
| `security_sensitive` | 보안/계정/금전 관련 우려 | 민감정보 입력 금지 + 공식 고객센터 안내 + human review |
| `fallback_general` | 분류 불가 또는 API 실패 | 일반 감사/사과 + 문의 경로 안내 |
