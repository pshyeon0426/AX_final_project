# 데모 스크립트 (10~15분) — 슈퍼SOL 고객 반응 분석

발표자용 진행 대본. 시간 배분과 멘트, 클릭 동선, 비상 대응(fallback)을 포함한다.

## 0. 사전 준비 (발표 전, 청중 앞 아님)
- [ ] `.env` 에 `OPENAI_API_KEY` / `OPENAI_MODEL=gpt-4o-mini` 설정 확인 (`python -m src.config` → `OPENAI_ENABLED True`)
- [ ] 데이터 준비: `data/processed/review_clean.csv` 존재. 없으면 `python -m src.pipeline` 1회 실행해 산출물 생성
- [ ] 앱 미리 띄워두기: `streamlit run app.py` → http://localhost:8501 접속
- [ ] 네트워크 불안 대비: **사전 수집 데이터 + 산출물**로 오프라인 시연 가능(수집 버튼 없이 진행)
- [ ] ⚠️ 화면 공유 시 `.env`/터미널에 **API 키 노출 금지**

## 1. 오프닝 & 문제 정의 (1.5분)
> "슈퍼SOL 출시 초기, 고객 반응을 빠르게 읽고 경쟁 앱과 비교하며, 부정 리뷰에 답글까지
> 만드는 과정을 **하나의 대시보드**로 자동화했습니다."
- 3대 기능 소개: **VOC 분석 · 경쟁 벤치마킹 · 답글 초안**
- 보안 원칙 한 줄: "실제 키·고객정보는 코드에 없고, 답글은 **자동 게시 아닌 검수용 초안**"

## 2. 데이터 & 수집 구조 (1.5분)
- 사이드바 **`1·데이터 소스`** 펼치기 → "앱별로 Google Play + App Store(공개 RSS) 직접 수집,
  결과는 **미리보기 후 적용**" 설명 (실제 수집은 시간상 생략, 구조만 보여줌)
- **`source_manifest.csv`** 언급: "어디서·어떻게 수집했는지 근거를 남깁니다"
- 한 줄: "실데이터가 없으면 11개 이슈를 포함한 **샘플 데이터로 fallback**"

## 3. 메인 대시보드 — VOC 한눈에 (2분)
- 상단 **컨텍스트 배지**(기준 앱·기간·파이프라인 PASS) 가리키기
- **KPI 4종**: 리뷰 수 · 평균 평점 · 부정률 · 토픽 수 (색으로 위험도 표시)
- **차트**: 리뷰 추이 / 감성 분포 / 불만 유형 TOP5 / 앱별 평점 비교
- 사이드바 **기간 슬라이더**를 좁혀 → KPI·차트가 **실시간 반영**되는 것 시연

## 4. 경쟁 벤치마킹 탭 (1.5분)
- "전체 앱 요약": **KB스타뱅킹 4.09 vs 슈퍼SOL 2.55(6개 중 4위)**
- 핵심 인사이트: "공통 최다 불만 = **오류** → 안정성이 차별화 포인트"
- 강조: "**공개 리뷰 기준만**, 경쟁사 내부 전략은 추정하지 않음"

## 5. 토픽 / 불만 유형 (1분)
- 토픽 탭: 10개 토픽(인증·업데이트·광고·UX 등), `topic_label` 검수 컬럼 설명
- "토픽=비지도 ML(KMeans), 불만 유형=룰(키워드 사전) — **단계가 다름**"

## 6. AI 리포트 (1.5분)
- AI 리포트 탭: 상단 배지 "**LLM(gpt-4o-mini)로 생성**" 확인(룰/LLM 표시는 실제 기록 기준)
- 6섹션 스크롤(핵심요약~데이터 한계)
- "프롬프트에 **근거 없는 추정·내부 전략 추정 금지** 명시 → 근거 리뷰를 함께 주입(경량 RAG)"
- (선택) **🔁 현재 필터 기준 재생성** 버튼 1회 시연

## 7. 답글 생성 — 핵심 시연 (2.5분)
- 답글 탭 → **우선순위순**으로 부정 리뷰 자동 정렬 확인
- 🎯 대상 리뷰 정보 카드(앱·별점·감성·심각도·이슈·원문) 보여주기
- **🧭 LLM 반영 지침**에 예: `고객센터 1599-8000 안내` 입력
- **"답글 생성/재생성"** → 생성된 초안에 **번호가 반영**되는지 확인
- 안전장치 강조: "보안/금전 이슈면 **검수 필요** 자동 표시, 금칙어는 후처리에서 차단"
- (선택) **직접 붙여넣기** 모드: 데이터에 없는 댓글을 붙여넣어 즉시 답글 생성

## 8. 검증 탭 (1분)
- 데이터 품질 PASS, 감성 baseline 0.547 → 개선 0.576
- **실패/리스크 케이스**: 감성 오분류 예시, 답글 안전성 위반 **0건** 보여주기

## 9. 마무리 — 한계 & 고도화 (1분)
- "감성은 별점 **약지도** 라벨의 한계 → 수동 검수로 개선 여지"
- "고도화: 벡터 RAG 근거 강화, 임베딩 감성 모델, 정기 자동 수집, 답글 검수 워크플로우"
- 한 줄 클로징: "**내부 데이터 없이 공개 리뷰만으로** 출시 초기 CX 의사결정을 지원합니다."

## 비상 대응
- **LLM 오류/지연**: 앱이 자동으로 룰 기반 fallback → "키 없거나 실패해도 멈추지 않습니다" 로 설명 전환
- **수집 실패**: 사전 산출물로 진행(수집 시연은 구조 설명으로 대체)
- **수치가 화면과 다름**: "수집 시점에 따라 변동" 안내. 발표 직전 `python -m src.pipeline` 로 산출물 동기화 권장
