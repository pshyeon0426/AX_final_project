## A. 상태 일관성 (LLM ↔ fallback 표시)

- **A1** [High] `.env` 에 유효 키 설정 → 앱 재시작 → 사이드바/AI리포트/답글탭 **세 곳 모두** "API 사용(gpt-4o-mini)" 로 일치하는가.
  - 기대: 세 화면의 상태 문구가 동일. 리포트 배지 = `LLM (gpt-4o-mini)`.
  - 현재: ❌ (F1) 캐시 때문에 불일치 가능.
- **A2** [High] 키 제거 → 재시작 → 세 곳 모두 "API 미사용/fallback" 로 일치.
- **A3** [Med] 단일 진실원본(SSOT): 화면의 "생성 방식"은 산출물(`performance_log`)의 실제 기록과 항상 일치해야 한다(설정값 추정 금지).

## B. 산출물 신선도 / 캐시 무효화

- **B1** [High] 데이터 업로드/교체 → KPI·차트·테이블이 **새 데이터로 갱신**되는가.
  - 기대: `load_*` 가 데이터 지문(파일 mtime/size)으로 무효화.
  - 현재: ❌ (F1) 파일명만 키 → 미갱신.
- **B2** [High] "전체 파이프라인 실행" 후 AI리포트/답글/검증 탭이 **방금 생성된 산출물**을 보여주는가.
  - 현재: ❌ 캐시 미무효화 시 옛 산출물 표시.
- **B3** [Med] "새로고침/캐시 비우기" 수단 제공(버튼 또는 자동).

## C. 기능 정합성

- **C1** [High] 사이드바 앱/기간/경쟁앱 필터 변경 → KPI·차트가 즉시 반영.
- **C2** [Med] 빈 데이터/단일 앱/날짜 결측 등 **엣지 케이스**에서 에러 없이 안내 메시지.
- **C3** [Med] 차트 수치 = 테이블/`metrics.json` 수치 일치(부정률, 리뷰 수 등).
- **C4** [Med] 토픽 `topic_label` 공란, 불만유형 TOP5 등 산출물 컬럼이 화면과 일치.

## D. 답글 안전성 · 정책

- **D1** [High] 보안/금전/심각 오류 리뷰 → `needs_human_review=True` + 민감정보 미요청.
- **D2** [High] LLM 답글도 생성 후 `safety_check_reply` 재검증 통과(금칙어/확정약속/투자조언/PII 요청 0건).
- **D3** [High] "자동 게시 아님 · 검수용 초안" 경고가 답글 탭에 항상 노출.
- **D4** [Med] `reply_risk_cases.csv` 위반 0건 유지(발표 기준).
- **D5** [Med] PII 포함 리뷰는 `[MASKED]` + 검수 플래그.

## E. UI/UX

- **E1** [Med] 답글 "재생성" 시 로딩 스피너 + 결과 즉시 반영(세션 상태).
- **E2** [Med] 답글 복사 버튼/CSV 다운로드 정상 동작.
- **E3** [Low] 한글/이모지/레이아웃 깨짐 없음(웹은 UTF-8이라 정상 예상, 점검).
- **E4** [Med] 30초 기준 안내 문구 노출("사전 학습/인덱스 준비 후 분석 요청 기준").
- **E5** [Low] 탭 전환·차트 인터랙션 반응성, 빈 탭 안내.

## F. 성능

- **F1p** [Med] 사전 산출물 로딩 기반 분석 응답 ≤ 30초(LLM 답글 생성 제외).
- **F2p** [Med] LLM 답글 1건 생성 지연 측정·표시(현재 ~2.2s/건).
- **F3p** [Low] 캐시 적용 후 재방문 시 즉시 로딩.

## G. 보안

- **G1** [High] 코드/문서/로그/산출물에 실제 키 **미포함**(grep 점검).
- **G2** [High] `.env` 가 `.gitignore` 로 보호, `.env.example` 만 공유.
- **G3** [Med] performance_log/리포트에 키·PII 미기록.
- **G4** [High] 노출된 키는 **폐기·재발급**(운영 수칙).

## H. 재현성 / 테스트

- **H1** [High] `pytest` 전체 통과(현재 49건).
- **H2** [Med] `run_pipeline()` 재실행 결정성(시드 고정 단계).
- **H3** [Med] 단계 실패 시 전체 중단 없이 fallback(예외 주입 테스트).

## 2. 우선 수정 권고 (발표 전)

1. **F1/B1/B2 (캐시 무효화)** — `load_*` 에 데이터 지문/파일 mtime 을 인자로 추가하거나
   `ttl` 설정, + 사이드바 "🔄 새로고침(캐시 비우기)" 버튼. → 사용자가 본 버그 직접 해결.
2. **A1/A3 (상태 SSOT)** — 화면의 LLM/fallback 표시는 `performance_log` 실제 기록만 근거로.
3. **F3 (설정 반영)** — `.env` 변경 시 재시작 안내 또는 설정 리로드 버튼.

## 3. 합격 기준 (발표 가능 수준)

- 상태 일관성 A1/A2 ✅, 캐시 신선도 B1/B2 ✅
- 답글 안전성 위반 0건(D), 검수 경고 노출(D3)
- pytest 전체 통과(H1), 키 미노출(G1/G2)
- 분석 응답 ≤ 30초(F1p)
